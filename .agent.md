# Agent Instructions - Skillijob React Project

## Project Overview

**Skillijob** is a modern React-based recruitment platform connecting job seekers with companies. The platform provides a dual-sided marketplace where candidates can submit CVs and companies can browse/unlock qualified profiles.

- **Tech Stack:** React 18.2.0, React Router v6, Framer Motion, Vite
- **Port:** 3000 (development)
- **Main Branch:** main
- **Working Directory:** `C:\Users\Utilisateur\Downloads\site internet`

### 1. Home (`/`)
- **File:** `src/pages/Home.jsx`
- **Purpose:** Main landing page
- **Sections:** Hero, Problem Statement, Sectors Grid
- **CTAs:** Redirect to Candidates or Companies pages

- **File:** `src/pages/Candidates.jsx`
- **Purpose:** Job seekers landing page
- **Key Features:**
  - CV submission via JotForm: `https://form.jotform.com/252881502955059`
  - Partners carousel (8 partners)
  - Why Skillijob section
  - How it works (3 steps)
  - FAQ section
  - CTA modal
- **Chatbot:** Eloise (Assistant Recrutement)

### 3. Companies (`/entreprises`)
- **File:** `src/pages/CompaniesNew.jsx`
- **Purpose:** Recruiters landing page
- **Key Features:**
  - Job posting form: `https://form.jotform.com/252922402753050`
  - Problems solved (6 pain points)
  - Benefits grid (4 KPIs)
  - Pricing section (990€ HT for 5 profiles)
  - About section with stats
- **Hero Buttons:**
  1. "Voir les candidats" → `/espace-candidats`
  2. "Déposer une offre d'emploi" → JotForm (external)
  3. "Obtenir mes 5 profils qualifiés" → `/paiements`
- **Chatbot:** Clément (Assistant Recrutement Entreprise)

### 4. Candidates Directory (`/espace-candidats`)
- **File:** `src/pages/EspaceCandidats.jsx`
- **Purpose:** Browse and filter candidate profiles
- **Features:**
  - Search bar
  - Sidebar filters:
    - Region (dynamic from JSON)
    - Sector
    - Experience (0-5, 5-10, 10-15, 15+ years)
    - Availability
    - Skills (multi-select)
  - Sorting (Recent, Experience, Name, Location)
  - View toggle (List/Grid)
  - Expandable candidate cards
  - Modal with full profile details
  - "Unlock Profile" button
- **Data Source:** `src/data/candidates.json` (~3000+ profiles)

### 5. CV Upload (`/cv`)
- **File:** `src/pages/CV.jsx`
- **Purpose:** Alternative CV submission form
- **Form Fields:** Identity, Address, Professional Info, CV Upload
- **Note:** Currently less used than JotForm integration

### 6. Payments (`/paiements`)

## Development Guidelines

### When Modifying Code

1. **Always read files before editing**
   - Use Read tool to check current content
   - Understand context and dependencies

2. **Component modifications**
   - Each component has matching `.css` file
   - Maintain consistent styling with design system
   - Use CSS variables from `variables.css`

3. **Page additions/modifications**
   - Update `App.jsx` routing
   - Create both `.jsx` and `.css` files
   - Follow existing naming conventions

4. **Responsive design**
   - Test at breakpoints: 640px, 768px, 1024px
   - Mobile-first approach
   - Use Framer Motion for animations

5. **External links**
   - Use `external` prop on Button component
   - Open in new tab for JotForm/Stripe links

### File Reading Strategy

- **Before editing ANY file:** Use Read tool first
- **For search tasks:** Use Grep tool with appropriate flags
- **For file discovery:** Use Glob tool with patterns
- **For multi-step tasks:** Use TodoWrite to track progress

### Common Modifications

#### Adding a new page
```javascript
// 1. Create page file: src/pages/NewPage.jsx + .css
// 2. Update App.jsx:
import NewPage from './pages/NewPage';
<Route path="/new-page" element={<NewPage />} />
// 3. Add navigation link in Navbar.jsx
```

#### Modifying ChatBot URLs
- Edit `getChatbotUrl()` function in `src/components/layout/ChatBot.jsx`
- Update URL and title for specific routes

#### Updating Candidate Data
- Edit `src/data/candidates.json`
- Follow existing schema structure
- Ensure all required fields are present

#### Changing Payment Links
- Update `handleUnlockCredit()` in `src/pages/Paiements.jsx`
- Replace Stripe URL with new payment link

## Styling Guidelines

### CSS Architecture
- Use CSS variables for colors, spacing, shadows
- Component-scoped CSS files (one per component)
- BEM-like naming convention
- Mobile-first responsive design

### Animation Guidelines
- Use Framer Motion for complex animations
- Use CSS transitions for simple hover states
- Keep animations under 0.5s for responsiveness
- Stagger animations for lists/grids

### Responsive Patterns
```css
/* Mobile first */
.component { /* base mobile styles */ }

@media (min-width: 640px) { /* tablet */ }
@media (min-width: 1024px) { /* desktop */ }
```

## Testing & Validation

### Before Committing
- [ ] Test all modified pages in browser
- [ ] Check responsive design at 3 breakpoints
- [ ] Verify external links open correctly
- [ ] Test form submissions (if applicable)
- [ ] Check console for errors
- [ ] Validate accessibility (ARIA labels, semantic HTML)

### Common Issues
- **Broken images:** Check file paths in `src/assets/images/`
- **Routing issues:** Verify route paths in `App.jsx`
- **External links not working:** Add `external` prop to Button
- **Animations laggy:** Check Framer Motion props, use `transform` instead of `left/top`

## Architecture Principles

1. **Component Reusability:** Use shared components from `src/components/shared/`
2. **Single Responsibility:** Each component does one thing well
3. **Props Over State:** Pass data down, lift state up when needed
4. **CSS Modularity:** Component-scoped styles with global design tokens
5. **Performance First:** Lazy loading, optimized images, memoization
6. **Accessibility:** ARIA labels, semantic HTML, keyboard navigation
7. **Mobile-First:** Design for mobile, enhance for desktop
